SECURITY

Last updated: April 14, 2026

Security is foundational to Arrakis. We build and operate enterprise infrastructure for industries where data integrity and confidentiality are non-negotiable — Energy, Manufacturing, Construction, and Defense. This page describes our security programme, compliance posture, and how to report vulnerabilities.

Our SOC 2 Type I audit and ISO 27001 certification are currently in progress. Compliance documentation and audit reports will be available to customers under NDA upon request.

Data Protection

  • AES-256 encryption at rest for all customer data
  • TLS 1.2+ encryption in transit for all communications
  • Encryption key management via AWS KMS
  • Customer data is logically isolated per tenant
  • Data residency options available (EU, UK, US)

Access Control

  • Role-based access control (RBAC) enforced at the platform level
  • Multi-factor authentication (MFA) required for all internal systems
  • Principle of least privilege applied to all service accounts and IAM roles
  • Privileged access managed via dedicated IAM tooling with time-bound sessions
  • All access changes are logged and subject to periodic review

Infrastructure & Operations

  • Hosted on AWS with infrastructure spanning EU and US regions
  • AWS GuardDuty enabled for continuous threat detection
  • AWS CloudTrail and Security Hub for audit logging and posture monitoring
  • VPC network isolation with private subnets for production workloads
  • Automated vulnerability scanning via AWS Inspector and Dependabot
  • Container image scanning on every build

Secure Development

  • All code changes require peer review before merging to production
  • Branch protection enforced across all production repositories
  • Automated security and dependency scanning in CI/CD pipeline
  • Penetration testing conducted periodically by independent third parties
  • Security requirements embedded in the software development lifecycle (SDLC)

Business Continuity

  • Business Continuity and Disaster Recovery (BCDR) plan maintained and tested annually
  • Automated database backups with point-in-time recovery
  • Multi-region architecture for critical platform components
  • Incident response plan with defined escalation paths and SLAs

People & Process

  • Security awareness training completed by all employees upon onboarding and annually
  • Background checks conducted for all employees
  • Defined information security policies reviewed and approved annually
  • Vendor risk assessments for all third-party service providers with data access

Responsible Disclosure

We take security vulnerabilities seriously and appreciate responsible disclosure from the security community.

If you believe you have discovered a security vulnerability in our platform or infrastructure, please report it to us privately before public disclosure.

Contact: security@arrakistechnologies.ai

We commit to: acknowledge receipt within 2 business days, provide a status update within 10 business days, and work with you to understand and remediate the issue before any public disclosure. We will not pursue legal action against researchers who act in good faith and follow this process.
